# 家人操作手冊（mango-hsu V2）

V2 已從 Google Sheet 改為網站後台。所有操作都在 **`/admin`** 進行，
不再需要打開 Sheet。對帳、出貨、商品上下架，全部在後台點按鈕。

本文記錄日常操作。Telegram 通知設定請看「附錄 A」。

---

## 一、登入後台

1. 打開瀏覽器（建議用手機 Chrome 或 Safari）
2. 進入 `https://mango-hsu.rayclaw-worker.workers.dev/admin`
   （或正式 domain 上線後告訴你的網址）
3. 輸入 email + 密碼登入
4. 第一次登入：系統會強制要求改密碼，請設一組你記得住的

**忘記密碼**：自己改不了，請 LINE 告訴 Ray，他會在後台幫你重設。

**權限分兩級：**
- `admin`（Ray）— 可以管理商品、編輯訂單、取消訂單
- `operator`（家人）— 只能標已付款、已出貨、印揀貨單

---

## 二、看一眼今天有什麼事

登入後就會看到後台首頁，三個數字：

- **總訂單** — 累積到現在所有單
- **待付款** — 還沒打勾「已付款」的單（家人對帳目標）
- **待出貨** — 已收到款但還沒寄出的單（家人寄貨目標）

下方會列出最近 5 筆訂單。點訂單編號就能看詳情。

---

## 三、新訂單進來 — 你會看到什麼

跟 V1 一樣，**Telegram 群組會收到一則訊息**：

```

王小明  0912345678

• 金煌芒果乾 1斤 × 3

小計 $1350 + 運費 $80 = 總計 $1430

收件地址：台中市...

🔗 https://mango-hsu.rayclaw-worker.workers.dev/status?id=M-20260519-001
```

新訂單也會立刻出現在後台「訂單列表」最上面，不再寫到 Sheet。

---

## 四、對帳 — 確認收到款

1. 打開網銀 App，看「近期交易」
2. 找備註欄寫著 `M-20260519-001-王小明` 的入帳
3. 開後台，左上角點「訂單列表」
4. 用 filter 切到「**待付款**」
5. 找到那筆訂單，點訂單編號進去
6. 按 **「標為已付款」**（綠色大按鈕）
7. 確認對話框按確定 — 系統會自動：
   - 紀錄誰在何時打勾（變更歷史）
   - 推一則 LINE 訊息給客人（如果他們綁過 LINE）

**技巧**：客人忘了寫備註就用金額＋姓氏猜。猜不到就 LINE 問客人。

---

## 五、出貨

### A. 一筆一筆出
1. 後台點「訂單列表」→ filter 切「**已付款待出貨**」
2. 點訂單編號進去
3. 在「物流單號」欄填宅配單號（不填也可以，但客人就看不到）
4. 按 **「標為已出貨」**

客人去 `/status` 頁就會看到更新；綁過 LINE 的會收到通知。

### B. 一次出一批（推薦）
產季尖峰一天可能十幾筆。用批次處理：

1. 訂單列表 filter 切「**已付款待出貨**」
2. 勾選要出貨的那幾筆（或勾「全選」）
3. 點 **「一鍵生揀貨單」** — 進到揀貨單頁
4. 按右上「列印 (A4)」直接送印表機 — 這張紙拿著按品項揀
5. 全部包好後，按返回鈕回訂單列表（你勾選的會還在）
6. 一筆一筆把宅配單號填進去後按「標為已出貨」
   - 或：如果你**還沒**有宅配單號（先打包再去 7-11/便利商店），
     可以先按 **「批次標已出貨」**（會勾選的全部標成已出，物流單號之後再補）

**補物流單號**：進那筆訂單詳情，「物流單號」可以重編 — 但這個目前要 admin 幫忙。

---

## 六、商品上下架（admin 才能做）

這個只有 Ray 能做。但流程紀錄一下：

1. 後台首頁 → 「商品管理」
2. 找到 SKU
3. 把「上架」checkbox 取消勾選 → 按「儲存」
4. 該 SKU 在客人端立刻顯示「已售完」

**重要**：**不要刪 SKU**。歷史訂單還連著它，刪掉會壞事。下架就好。

新增 SKU：頁面下方「新增商品」表單填一填即可。

---

## 七、整季結束 — 暫停整個網站

V2 的「接單中／停接」開關目前是 worker 的環境變數 `ACCEPTING_DRY`，
不是後台 UI。要改的話請 LINE Ray，他大概兩分鐘可以重 deploy。

V3 之後會把這個搬到後台一個按鈕，目前還沒做。

---

## 八、客人要改訂單

V2 後台支援 admin 編輯地址、客戶備註、品項數量，以及取消訂單。但 operator 不能。

**operator（家人）**：請 LINE 告訴 Ray，他會幫你改。
**admin（Ray、太太）**：訂單詳情頁可改：
- 右下「admin: 編輯 / 取消」展開 → 改地址/備註，或取消整張單
- 中間「品項」區塊下面「admin: 編輯品項」展開 → 改某品項數量、刪除某品項、新增品項
- **只有「待付款 + 待出貨」狀態能改**。已付款／已出貨／已取消的單，按鈕不會出現。
  - 已付款要退？需走退款流程（不在 V4 範圍）。
  - 已出貨想取消？已經出去的貨叫不回來，請聯絡客人協調。

取消訂單會自動把那單佔的庫存還回去，給其他客人。

---

## 九、姑姑接到 LINE 訂單，我要灌進系統（V4）

姑姑那邊客人是用 LINE 私訊她，她接了單再 LINE 跟我們講。要把這些單也灌進系統，
庫存才不會超賣。

1. 後台「訂單列表」上方有一顆橘色按鈕 **「+ 代客建單」**
2. 進去後填：客人姓名 / 手機（09 開頭 10 碼）/ 地址 / 品項數量 / 備註
3. 每個品項旁邊會顯示**目前剩多少包**——選量時看清楚
4. 按「建立訂單」
5. 系統會：
   - 從那個 SKU 的庫存扣對應數量（自動）
   - 跳到那筆訂單詳情頁
   - **不**送 Telegram 通知（這單是我們自己建的，不需要再通知自己）
   - **不**推 LINE 訊息給客人（姑姑接到的客人本來就用 LINE 跟姑姑聯絡）

**雙擊／網路慢按了兩次的防護**：系統會把同一次表單的兩次送出當成同一張訂單，
不會建出兩筆、不會扣兩次庫存。

**規則：每天晚上對帳前，姑姑當天接的單都要建進來。** 否則庫存會跟現實對不上。

如果 LINE 上看姑姑的訊息少了某個欄位（如沒寫地址），LINE 問清楚再建。

## 十、庫存對帳 / 補貨（V4）

「商品管理」每個 SKU 旁邊現在會顯示**目前庫存幾包**，並有「調整」按鈕。

### 補貨（產季新進貨）
1. 商品管理 → 找到那個 SKU → 點「庫存 N 包 ・ 調整」展開
2. 把「新庫存」改成新的數字（例如本來剩 3 包，新進貨 50 包，就填 53）
3. 「原因」一定要填（例如「補貨：今天從爸爸果園拿到 50 包」）
4. 按「儲存」

系統會把這筆調整寫進變更紀錄，誰、什麼時候、從多少改到多少、原因是什麼，全部留底。

### 校正（盤點發現少 / 多 / 壞貨報廢）
跟補貨一樣，把「新庫存」設成實際數字、原因填清楚。

### 「未出貨單比新庫存多」的警告
如果你把庫存改得比目前已接但還沒出貨的單還少，系統會跳出 confirm 視窗：
「未出貨單共 X 包，新庫存 Y 比這還少，確定？」

意思是：你正在製造一個「客人已下單但實際物理上不夠出」的狀態。
真要這樣（例如壞貨報廢）就按確定，系統會留 override 紀錄；要重新想就按取消。

## 十一、看已取消的單

「訂單列表」頁面上方 filter 多了一個「**已取消**」分頁，點下去就看得到所有取消過的單。
取消單在預設「全部」分頁不會出現。

## 十二、變更紀錄（Audit）

後台首頁「變更紀錄」會列出所有人改過的所有事 —
誰幾點打勾、誰改了哪張單、誰登入失敗。出問題時很好查。

每張訂單詳情頁底下也有專屬的變更歷史。

---

## 十三、常見問題

### 我沒收到 Telegram 通知
- 確認 bot 還在群組裡
- 進「變更紀錄」搜 `telegram_failed` 看有沒有失敗紀錄
- 拿手機開 `/admin`，如果看得到那筆訂單，代表訂單有進系統，只是通知掉了
- 都沒解決就 LINE Ray

### 客人說下不了單／網站當掉
- 自己拿手機試試 `/order` 頁，重現一次
- 截圖傳給 Ray，附上錯誤訊息
- 進「變更紀錄」看最近有沒有怪東西

### 我點「標為已付款」但客人沒收到 LINE 通知
- 客人沒綁過 LINE OA 就不會收到（綁是客人從 `/status` 頁去做的）
- 月份累計推播超過 200 則會擋（不太會碰到）

## 附錄 A：Telegram 通知 setup（一次性）

V1 / V2 共用同一個 Telegram 機制，第一次設定才需要做。

1. 手機 Telegram 搜 `@BotFather` → `/newbot` → 取名 → 拿 token
2. 建一個 Telegram 群組「芒果許訂單」，把家人 + bot 都加進來，bot 設管理員
3. 群組裡發一則訊息（如「test」）
4. 瀏覽器打開 `https://api.telegram.org/bot<TOKEN>/getUpdates`，
   找到 `chat.id`（負數，開頭通常 `-100`）
5. 把 token 跟 chat id 給 Ray，他用 `wrangler secret put` 寫進系統：
   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_CHAT_ID`
6. 部署後測一張單（自己下訂），群組要收到通知才算過

**bot token 不小心外流**：回 `@BotFather` 傳 `/revoke` 換新 token，
再把新 token 給 Ray 重設 secret。
